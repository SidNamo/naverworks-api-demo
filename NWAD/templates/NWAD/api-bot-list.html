Naver Works Developer Console 접속
API를 연동하기 전에, 먼저 Naver Works Developer 페이지에서 API를 생성해야 합니다.

Naver Works Develper Console 링크
1. 로그인 및 Developer Console 접속
2. API 생성 및 이름 입력
※ 생성할 때 입력한 API 이름을 하단 입력칸에 입력하세요.
3. 생성한 API 정보 확인
4. 생성된 API의 Client ID 및 Client Secret 확인
※ 하단 입력칸에 동일하게 입력하세요.
5. Service Account 발급
6. 발급된 Service Account 확인
※ 하단 입력칸에 동일하게 입력하세요.
7. Private Key 발급
8. 발급한 Private Key 확인 및 복사
※ 하단 입력칸에 복사한 값을 붙여넣기 하세요.
9. Scope 설정
※ 이미지와 동일하게 bot.read, user.read는 필수로 적용하세요.
※ 하단 입력칸에 엔터로 구분하여 추가한 scope를 입력하세요.

Bot 연동하기

{% csrf_token %}
Naver Works Developer Console 접속
Bot을 연동하기 전에, 먼저 Naver Works Developer 페이지에서 Bot을 생성해야 합니다.

Naver Works Develper Console 링크
1. 로그인 및 Developer Console 접속
2. Bot 등록 버튼 클릭
3. Bot 정보 입력 - 하단과 동일하게 입력하지 않을 경우, 기능 사용에 문제가 발생합니다.
※ API 2.0 체크
※ Callback URL On 체크
※ Callback URL 주소 동일하게 입력 (고정) "https://apisimpler.didim365.com/callback"
※ Callback URL 구성원이 Bot에 보낼 수 있는 항목 중 "TEXT"만 체크
※ 팀/그룹, 1:N 메시지방 초대 가능 체크
4. 등록한 Bot 정보 확인
※ 생성된 Bot 정보를 하단 입력칸에 입력하세요.
2. 등록한 Bot을 사용중으로 상태 변경
Bot을 사용중으로 변경하기 위해서는 Naver Works Admin 페이지에 접속해야 합니다.

Naver Works Admin 링크
3. 기존에 등록한 API를 선택한다.
※ API를 등록하지 않았다면 우선 API를 등록해야 한다.
※ 선택한 API의 경우 입력한 Bot이 정상적인 상태인지 확인하는데만 사용이 된다.
